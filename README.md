# soonpung (순풍)

> **2026 서울시 빅데이터 활용 경진대회 — 창업 부문 출품작**

산후조리원 예약, AI 산후우울증 진단, 산후관리사 매칭, 정부 지원금 안내를 한 앱에서 해결하는 **산후 통합 플랫폼**입니다.

---

## 왜 이 서비스가 필요한가

산모가 출산 후 겪는 현실적인 문제가 세 가지 있습니다.

**첫째, 산후조리원 정보가 너무 파편화되어 있습니다.**
서울에만 산후조리원이 수십 곳인데 가격·시설을 한눈에 비교할 곳이 없습니다. 전화를 일일이 돌리거나 카페 글을 뒤적이는 게 현실입니다. 호텔은 야놀자로 예약하는데 산후조리원은 왜 아직도 전화 예약인가에서 출발했습니다.

**둘째, 산후우울증이 방치되고 있습니다.**
서울 산모의 10~15%가 산후우울증을 경험하지만, 현재 체계는 이미 증상이 심각해진 뒤 1366에 전화하는 사후 대응뿐입니다. 매일 1분 음성 일기로 사전에 감지하는 시스템이 없습니다.

**셋째, 받을 수 있는 지원금을 모릅니다.**
임산부 교통비, 첫만남이용권, 부모급여 등 중앙정부·서울시·자치구 지원금이 88가지나 되지만 어디서 신청하는지 아는 산모가 거의 없습니다.

## 핵심 기능 6가지

### 1. 홈 대시보드

현재 임신 주차를 자동 계산해 오늘의 할 일(음성 일기, 산후조리원, 지원금, 주차별 정보)을 한 화면에 보여줍니다. 가장 최근 PHQ-9 점수가 위험 수준이면 홈에서 바로 경고합니다.

### 2. 산후조리원 검색·비교·예약 (`/postpartum`)

서울시 산후조리업 인허가 공공데이터(OA-16482)를 기반으로 영업 중인 조리원만 필터링합니다.

- 자치구 25곳 필터 + 예산(400만 이하 / 400~600만 / 600만 이상) + 평점·가격·후기 정렬
- 시설 정보, 2주 기준 가격 범위, 예약 문의 버튼 제공

### 3. 음성 일기 + AI 산후우울증 진단 (`/diary`)

매일 1분 녹음만 하면 AI가 자동으로 우울 수준을 측정합니다.

```
사용자 음성 녹음
    ↓
OpenAI Whisper API (한국어 STT)
    ↓
GPT-4o-mini (PHQ-9 9개 항목 자동 분류)
    ↓
점수 + 위험등급 + 위기 키워드 감지
    ↓
점수 ≥ 10점 → 1366 즉시 연결 알림
```

**PHQ-9 항목** (각 0~3점, 총 27점):
흥미 저하 / 우울감 / 수면 문제 / 피로감 / 식욕 변화 / 자존감 저하 / 집중력 저하 / 행동 변화 / 자해 사고

위험 등급: `정상(0~4)` / `경증(5~9)` / `중등도(10~19)` / `중증(20~27)`

PHQ-9 항목 9번(자해 사고)이 1점 이상이면 즉시 중등도 이상으로 격상됩니다.

텍스트 입력도 지원하므로 마이크가 없거나 말하기 불편한 상황에서도 사용할 수 있습니다.

### 4. 임신주수별 맞춤 콘텐츠 (`/content`)

서울시 모자보건사업 통계(dataList/41) 기반으로 현재 주차에 받아야 할 검진, 교육, 영양제 정보를 자치구별로 안내합니다.

### 5. 지원금 자동 매칭 (`/support`)

몽땅정보 만능키(OA-22188)의 88개 정책을 임신 주차 기준으로 자동 필터링합니다. 지금 신청 가능한 지원금 총액을 바로 계산해 보여주고, 카테고리별로 정리하여 자격 조건·신청 방법·바로가기 링크까지 제공합니다.

### 6. 긴급 SOS (`/sos`)

서울시 여성긴급전화 1366 운영실적(dataList/65)과 성·가정폭력 상담소 데이터(data.go.kr) 기반으로 현재 위치 기준 가장 가까운 상담소를 거리순으로 보여줍니다. 1366 원터치 전화 연결, 상담 유형별 통계도 제공합니다.

## 실행 방법

### 사전 조건

- Node.js 18 이상
- OpenAI API 키 (없으면 mock 데이터로 동작)

### 설치 및 실행

```bash
# 1. 패키지 설치
cd postmom
npm install

# 2. 환경변수 설정
cp .env.local.example .env.local
```

`.env.local` 파일을 열어 아래 값을 입력하세요:

```env
OPENAI_API_KEY=sk-...          # OpenAI API 키
SEOUL_OPENAPI_KEY=...          # 서울 열린데이터광장 키 (선택)
```

```bash
# 3. DB 마이그레이션 (최초 1회)
npx drizzle-kit push

# 4. 시드 데이터 입력 (최초 1회)
npx tsx src/scripts/seed.ts

# 5. 개발 서버 실행
npm run dev
```

`http://localhost:3000` 접속

> OpenAI API 키가 없어도 mock 데이터로 PHQ-9 채점이 동작합니다.
