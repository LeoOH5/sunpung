## 1. 홈 대시보드

### 한 줄 설명
앱을 켜면 지금 몇 주차인지 자동 계산해서 오늘 해야 할 일을 한 화면에 보여줍니다.

### 화면에서 보이는 것
- **임신 주차 원형 게이지** — 출산예정일 기반으로 현재 몇 주차인지, 40주 대비 몇 %인지 실시간 계산
- **D-day 카운터** — 출산예정일까지 남은 날
- **오늘의 할 일 4가지** — 아기수첩 / 산후조리원 / 지원금 / 이번 주 정보
- **아기수첩 최근 기록** — 마지막으로 기록한 일지 제목 + 총 기록 건수 표시
- **긴급 SOS 배너** — 하단에 항상 고정
- **활용 공공데이터 태그** — 7개 데이터셋 이름 표시

### 계산 로직
```
현재 임신 주차 = (280일 - 출산예정일까지 남은 일수) ÷ 7
```
예: 출산예정일 2026-08-15 기준으로 오늘(2026-05-12) → 약 28주차

### 활용 데이터셋
| 데이터 | 용도 |
|---|---|
| 산후조리업 인허가 (OA-16482) | "서울 N곳 비교·예약" 카드의 조리원 수 |
| 몽땅정보 만능키 (OA-22188) | "N개 정책 자동 매칭" 카드의 정책 수 |
| 아기수첩 DB (자체) | 최근 기록 제목 + 총 기록 건수 |

## 2. 산후조리원 검색·비교·예약

### 한 줄 설명
서울시 공공데이터로 영업 중인 산후조리원만 불러와 자치구·예산·평점 기준으로 비교하고 예약 문의까지 연결합니다.

### 화면에서 보이는 것
- **자치구 25개 필터 탭** (강남구~중랑구 전체)
- **예산 필터** — 400만 이하 / 400~600만 / 600만 이상 / 전체
- **정렬** — 평점순 / 가격순 / 후기순
- **카드 목록** — 조리원명, 주소, 평점, 후기 수, 시설 태그, 2주 기준 가격 범위, 예약 문의 버튼

### 핵심 로직
1. DB에서 `businessStatus = '영업'` 인 조리원만 필터링 (폐업 제외)
2. 자치구 선택 시 해당 구만 추가 필터
3. 예산 선택 시 `priceMin` 범위 조건 추가
4. 정렬 기준에 따라 `rating DESC` / `priceMin ASC` / `reviewCount DESC` 정렬

### 활용 데이터셋
**서울시 산후조리업 인허가정보 — OA-16482**

| 항목 | 원본 데이터 컬럼 | 앱에서 활용 |
|---|---|---|
| 시설명 | 사업장명 | 카드 제목 |
| 주소 | 도로명주소 | 위치 표시 |
| 자치구 | 소재지_구 | 자치구 필터 기준 |
| 영업 상태 | 영업상태명 | 폐업 필터링 |
| 정원 | 영업장면적 | 규모 파악 |
| 허가 날짜 | 인허가일자 | 영업 연차 계산 |

> 가격·평점·시설 정보는 공공데이터에 없어서 시드 데이터로 보완

### 비즈니스 모델 연결
산후조리원 평균 이용금액 **350만원**, 수수료율 **5~8%** = 건당 **17~28만원** 수익

## 3. 아기수첩 (육아일지 + AI 위험 감지)

### 한 줄 설명
사진·날짜·내용으로 아기와의 소중한 순간을 기록하고, AI가 글 내용에서 우울증·아동학대 징후를 감지하면 즉시 관련 기관으로 연결합니다.

### 화면에서 보이는 것
- **기록 목록 (페이지당 6건)** — 사진 썸네일 + 제목 + 날짜 + 내용 미리보기 카드
- **페이지네이션** — 이전/다음 + 페이지 번호 버튼
- **위험 뱃지** — 우울의심(주황) / 학대의심(빨강) 자동 태그
- **새 기록 폼 (전체화면)** — 사진 첨부 / 날짜 / 제목 / 내용 입력
- **상세 보기** — 사진 전체 + 내용 전문 + 위험 경고 + 전화 버튼
- **즉시 연결 버튼**
  - 우울 의심 → **1336** (정신건강위기상담) + **1393** (자살예방상담전화)
  - 학대 의심 → **112** (긴급신고) + **1366** (여성긴급전화)

### AI 처리 흐름

```
① 사용자가 일지 내용 입력 후 저장 버튼
         ↓
② POST /api/journal (제목 + 날짜 + 내용 + 이미지)
         ↓
③ GPT-4o-mini에 내용 전달
   우울증·아동학대 위험 키워드 감지 요청
         ↓
④ 판정 결과
   ├─ 정상 → 저장 후 목록 갱신
   ├─ 우울의심 → 저장 + 1336/1393 연결 버튼 노출
   └─ 학대의심 → 저장 + 112/1366 연결 버튼 노출
         ↓
⑤ DB 저장 (제목·내용·날짜·이미지·위험등급·alertType)
```

### 이미지 처리 방식
```
사용자 사진 선택 (카메라 또는 갤러리)
         ↓
브라우저 Canvas API로 클라이언트 사이드 압축
  - 최대 800×800px 리사이즈
  - JPEG quality 0.75
  - 평균 50~150KB 수준으로 압축
         ↓
Base64 Data URL로 변환 → DB에 직접 저장
(별도 스토리지 서버 불필요, 서버리스 환경 최적)
```

### AI 위험 감지 기준

| 감지 유형 | 판정 기준 예시 | 연결 기관 |
|---|---|---|
| **우울의심** | "죽고 싶다", "너무 힘들다", "아무것도 하기 싫다", 극도의 절망·자해 표현 | 1336, 1393 |
| **학대의심** | 아기에게 폭력·방치, "때리고 싶다", 과도한 체벌 언급 | 112, 1366 |
| **정상** | 위 기준에 해당하지 않음 | — |

### GPT 프롬프트 핵심 내용
```
아래 육아일지 내용을 분석해 위험 신호를 감지하라.

감지 기준:
- "우울의심": 산모의 우울증, 자해·자살 충동, 극도의 절망감
- "학대의심": 아동에 대한 폭력, 방치, 아기를 해치고 싶다는 표현
- "정상": 위 기준에 해당하지 않음

출력 형식: JSON
{
  "riskLevel": "정상|우울의심|학대의심",
  "alertType": "1336|112|null",
  "triggers": []
}
```

> alertType이 null이 아닌 경우, 저장 즉시 화면에 경고 배너 + 전화 버튼 노출

### 실제 예시

**예시 1 — 우울의심 감지**
- 입력: *"오늘도 너무 힘들었어. 애가 울면 나도 같이 울고 싶어. 사라지고 싶다는 생각이 자꾸 들어."*
- GPT 판정: `riskLevel: "우울의심"`, `alertType: "1336"`
- 화면: 주황 경고 + "1336 정신건강위기상담" + "1393 자살예방상담전화" 버튼

**예시 2 — 학대의심 감지**
- 입력: *"밤새 울어서 진짜 미칠 것 같았어. 아이를 때리고 싶은 충동이 들었다."*
- GPT 판정: `riskLevel: "학대의심"`, `alertType: "112"`
- 화면: 빨간 경고 + "112 긴급신고" + "1366 여성긴급전화" 버튼

### 활용 데이터셋
**서울시 여성긴급전화 1366 운영실적 — dataList/65**

| 데이터 내용 | 활용 방식 |
|---|---|
| 월별 상담 건수 (가정폭력·성폭력·스토킹 등) | 위험 유형 분류 기준 및 통계 근거 |
| 상담 유형 분류 | SOS 화면 통계 시각화 |
| 서울 지역 산후 위기 통계 | 학대·우울 위험 베이스라인 |

> 1366 데이터의 "가정폭력 상담이 산후 3개월 내 집중된다"는 통계가 이 기능의 도입 근거

### 비즈니스 모델 연결
- B2G: 자치구 산후우울증·아동학대 조기 감지 사업 위탁 → **자치구당 연 5,000만~1억**
- SaMD: 식약처 디지털 의료기기 2등급 등록 시 보험 수가 적용 가능

## 4. 임신주수별 맞춤 콘텐츠

### 한 줄 설명
현재 임신 주차에 해야 할 검진·교육·지원을 자치구 보건소 데이터와 조산 위험 통계로 개인화해서 보여줍니다.

### 화면에서 보이는 것
- **주차별 가이드 카드** — 이번 시기에 꼭 해야 할 3가지 (예: 28주 → 임신성 당뇨 검사)
- **임신주수별 출생 분포 차트** — 조산·만삭 비율을 시각화하여 현재 주차의 위험도 인지
- **자치구 보건소 프로그램 목록** — 현재 주차에 해당 자치구에서 신청 가능한 검진·교육·지원

### 핵심 로직
1. 현재 임신 주차 계산 (출산예정일 기반)
2. DB에서 `weekRangeStart ≤ 현재주차 ≤ weekRangeEnd` AND `district = 사용자자치구` 조건으로 프로그램 필터
3. 가장 가까운 주차 가이드 자동 선택 (12/16/20/24/28/32/36/40주 기준)

### 활용 데이터셋

**① 서울시 모자보건사업 통계 — dataList/41**

| 데이터 내용 | 활용 방식 |
|---|---|
| 자치구별 산전·산후 검진 프로그램 | 현재 주차+자치구 기준 자동 필터 |
| 출산준비교실, 영양제 지급 실적 | 신청 가능한 교육 프로그램 안내 |
| 수혜자 수 | 인기도 표시 |

> 예: 마포구 거주 28주 산모 → "마포구 보건소 임신성 당뇨 검사 교육 신청 가능" 자동 안내

**② 서울시 임신기간별 출생 통계 — dataList/10872**

| 데이터 내용 | 활용 방식 |
|---|---|
| 주수 구간별 출생 건수 (28주 미만 / 28~31주 / 32~36주 / 37주 이상) | 조산 위험도 시각화 |
| 연도별 조산율 | 위험 등급 색상 표시 (빨강/주황/노랑/초록) |

> 예: "37주 이상 출생이 94.3%" → 안전 구간임을 막대 그래프로 직관적으로 표시

**③ 데이터 결합 방식**
```
Dataset 2 (모자보건사업) + Dataset 6 (임신주수별 출생)
→ "이번 주차에 받아야 할 검진" + "지금이 얼마나 위험한 구간인지"
→ 두 가지를 한 화면에서 통합 제공
```

## 5. 지원금 자동 매칭

### 한 줄 설명
정부·서울시 88개 지원금 정책을 현재 임신 주차 기준으로 자동 필터링하고, 지금 당장 받을 수 있는 금액 합계를 즉시 보여줍니다.

### 화면에서 보이는 것
- **총 수령 가능 금액** — 지금 신청 가능한 정책의 금액 합산 (예: "약 330만원")
- **매칭된 정책 수** — 현재 주차 기준으로 해당하는 정책 수
- **카테고리별 분류** — 임신 / 출산 / 육아 / 주거 / 돌봄
- **각 정책 카드**
  - 지원 기관 (서울시 / 보건복지부 / 자치구)
  - 정책명·설명
  - 지원 금액 뱃지
  - 자격 조건
  - 신청 방법
  - 바로 신청 링크

### 핵심 로직
```
현재 임신 주차 계산
    ↓
DB에서 applicableWeekStart ≤ 현재주차 ≤ applicableWeekEnd 필터
    ↓
해당 정책 목록 반환
    ↓
금액 텍스트에서 숫자 파싱 (정규식) → 총액 합산
    ↓
카테고리별 그룹핑 후 화면 표시
```

### 활용 데이터셋

**서울시 몽땅정보 만능키 OpenAPI — OA-22188**

몽땅정보 만능키(umppa.seoul.go.kr)는 2024년 9월 오픈한 서울시 공식 임신·출산·육아 통합 정책 포털로, 첫 해 140만 방문자를 기록했습니다.

| 데이터 내용 | 앱에서 활용 |
|---|---|
| 중앙정부 21개 + 서울시 67개 = **총 88개 정책** | 전체 정책 DB로 활용 |
| 정책명, 지원 기관, 지원 금액 | 카드 표시 |
| 자격 요건 (임신주수, 소득, 자녀수) | 필터링 조건 |
| 신청 방법 (온라인/방문) | 신청 가이드 |
| 신청 URL | 바로 신청 버튼 |
| 해당 자치구 한정 여부 | 자치구 매칭 |

**대표 포함 정책 예시**
| 정책 | 금액 | 신청 시기 |
|---|---|---|
| 임산부 교통비 지원 | 70만원 (서울시) | 임신 확인 후 |
| 첫만남이용권 | 200만원 (국가) | 출산 후 |
| 부모급여 | 월 100만원 (국가) | 출산 후 |
| 산모·신생아 건강관리 바우처 | 최대 180만원 | 출산 예정 4주 전 |
| 영아수당 | 월 30만원 (서울시) | 출생 후 |

> 이 정책들을 일일이 검색할 필요 없이 주차만 입력하면 자동으로 나옵니다.

### 비즈니스 모델 연결
지원금 정보로 사용자 **월간 재방문율(MAU 유지)** 확보 → 다른 BM(OTA, 보험)의 베이스 트래픽

## 6. 산후관리사 매칭

### 한 줄 설명
출산 후 집에서 신생아·산모를 케어해주는 산후관리사를 자치구·경력·전문 분야 기준으로 매칭해서 예약 연결합니다.

### 화면에서 보이는 것
- **정부 바우처 안내 배너** — 기준중위소득 150% 이하이면 정부 지원 가능 링크
- **관리사 카드 목록** — 이름, 사진, 경력 연수, 자치구, 평점, 후기 수, 전문 분야 태그, 자격증 배지, 시간당 금액, 매칭 신청 버튼

### 핵심 로직
- 사용자 자치구(마포구)와 같은 관리사를 먼저 정렬
- 같은 자치구 내에서는 평점 내림차순 정렬

### 활용 방식
- 보건복지부 산모·신생아 건강관리 서비스 바우처 연계 안내
- 향후 실제 관리사 DB 연동 시 서울시 출생·사망 통계(dataList/10593)로 수요 예측

### 비즈니스 모델 연결
관리사 평균 80시간 이용 기준 **200만원**, 수수료 10% = **건당 20만원** 수익

## 7. 긴급 SOS

### 한 줄 설명
위기 상황에서 1366에 원터치 전화를 연결하고, 사용자 위치 기준 가장 가까운 상담소를 거리순으로 즉시 보여줍니다.

### 화면에서 보이는 것
- **1366 즉시 전화 버튼** — 빨간 카드, 탭하면 바로 전화
- **가까운 상담소 5곳** — 거리 km 순서, 24시간 여부 뱃지, 주소, 전화 버튼
- **서울시 1366 상담 유형 통계** — 가정폭력/성폭력/스토킹/데이트폭력 등 건수 막대 그래프

### 핵심 로직

**거리 계산 (Haversine 공식)**
```
두 좌표 사이의 실제 거리(km) 계산
→ 사용자 위치(위도·경도) 기준 상담소 전체 정렬
→ 가장 가까운 5곳 표시
```

**음성 일기와 연동 (향후)**
```
음성 일기에서 위기 키워드 감지
  예: "맞았어요", "죽고 싶어요", "도망가고 싶어요"
    ↓
triggers 배열에 키워드 저장
    ↓
PHQ-9 점수 ≥ 10 또는 triggers 존재 시
    ↓
SOS 화면으로 안내 + 1366 연결 버튼 강조
```

### 활용 데이터셋

**① 서울시 여성긴급전화 1366 운영실적 — dataList/65**

| 데이터 내용 | 앱에서 활용 |
|---|---|
| 월별 상담 건수 | 상담 유형 통계 차트 |
| 가정폭력·성폭력·스토킹·데이트폭력·디지털성범죄·성매매 분류 | 항목별 막대 그래프 |

> 이 통계를 보여주는 것만으로도 "산후에 왜 이 앱이 필요한지" 설명됨 — 산후 시기에 가정폭력이 급증하는 패턴이 데이터로 보임

**② 서울시 성폭력·가정폭력 상담소 정보 — data.go.kr/28453071**

| 데이터 내용 | 앱에서 활용 |
|---|---|
| 상담소명, 유형 (성폭력/가정폭력/통합) | 카드 표시 |
| 주소, 전화번호 | 연락처 연결 |
| 위도·경도 | 거리 계산 (Haversine) |
| 24시간 운영 여부 | 24시간 뱃지 |
| 운영시간 | 방문 가능 여부 |

**③ 두 데이터 결합 방식**
```
Dataset 3 (1366 실적) + Dataset 4 (상담소 위치)
→ "서울에서 이런 위기가 이만큼 일어나고 있고"
   + "지금 당신 근처 N km 안에 상담소가 있습니다"
→ 통계 맥락 + 즉각적 행동 연결
```

### 비즈니스 모델 연결
B2G 핵심 근거: 서울시 1366 데이터를 직접 연계한 민간 앱이 없음 → 자치구 위탁 사업 진입 가능
